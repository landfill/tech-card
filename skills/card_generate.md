# Card Generate

입력된 **레터 마크다운(letter_md)** 과 **날짜(date, YYYY-MM-DD)**를 바탕으로, **카드뉴스 슬라이드용 JSON** 한 개만 출력하라.
배경 이미지는 1호당 1장으로, 모든 카드가 공유한다. 카드 수는 5~8장으로 구성한다.

## 필수 출력 형식 (JSON만 출력)

```json
{
  "date": "YYYY-MM-DD",
  "cards": [
    {
      "type": "cover",
      "title": "한 줄 제목",
      "body": ["문장1", "문장2"]
    },
    {
      "type": "highlight",
      "title": "이번 호 핵심 소식",
      "body": ["요약 문장1", "문장2"]
    },
    {
      "type": "news",
      "title": "소식 제목",
      "body": ["내용 문장1", "문장2"]
    },
    {
      "type": "insight",
      "title": "인사이트 제목",
      "body": ["포인트1", "포인트2"]
    },
    {
      "type": "closing",
      "title": "마무리 제목",
      "body": ["마무리 문장1", "문장2"]
    }
  ]
}
```

## 카드 타입

| type | 설명 | 개수 |
|------|------|------|
| cover | 표지. 레터 헤드라인·날짜·테마 키워드 활용 | 1 |
| highlight | 하이라이트. 오늘 가장 중요한 소식 1개 요약 | 1 |
| news | 주요 소식. 레터 본문 섹션에서 선별한 소식 1~3장 | 1~3 |
| insight | 인사이트. 에디터 관점·트렌드 요약 | 1 |
| closing | 마무리. 다음 호 예고·독자 인사 | 1 |

## 규칙

- **cards** 배열만 사용. 그 외 키는 date만 허용.
- **body**는 문자열 배열. 한 요소당 한 문장 또는 한 줄. 2~5개 권장.
- **title**은 1~2줄 분량으로 짧게.
- 레터에 없는 내용을 만들지 말 것. 레터 본문만 요약·선별하여 사용.
- 코드 블록 마크업 없이 **JSON만** 출력하라.
