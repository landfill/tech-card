# Letter Generate

입력된 항목(items)과 날짜(date, YYYY-MM-DD)를 바탕으로 **데일리 인텔리전스 뉴스레터** 본문을 마크다운으로 작성하라.
**아래 items에 있는 항목만 사용할 것. items에 없는 사건·발표·URL은 서술하지 말 것. 모르는 내용을 추론하거나 보강하지 말 것.**

## 필수 구조 (반드시 준수)

### 1. 헤드라인 (첫 줄)
- 형식: `YYYY년 M월 D일 IT/AI 데일리 인텔리전스: '(오늘의 테마 키워드)' (한 줄 제목)`
- date를 한글 날짜로 바꾸고, 오늘 수집된 내용을 관통하는 테마를 한 문장으로 제목화하라. 예: `2026년 2월 25일 IT/AI 데일리 인텔리전스: '클라우드 에이전트'가 여는 비동기 개발 혁명`
- 모든 타이틀(헤드라인 포함) 뒤에는 반드시 한 줄의 빈 줄을 넣어, 다음 문단(오프닝 또는 요약)이 다음 줄에서 시작되도록 하라.

### 2. 오프닝 단락 (한 단락)
- "오늘 테크 생태계의 핵심은 …" 식으로 **2~4문장**의 서술형 도입. 장소·시간·역할의 변화, 트렌드 한 줄 요약을 담아 독자가 하루 요약의 맥락을 잡게 하라. 불릿·목록 금지.
- 오프닝 단락은 헤드라인 다음 줄(빈 줄 후)에 시작되어야 한다.

### 3. 본문 섹션 (1~4)
- **동일 주제 묶기 (필수)**: **같은 제품·이벤트·주제에 속한 항목은 반드시 한 덩어리(한 카테고리)로 묶어서** 서술하라. 한 주제가 여러 섹션에 흩어지거나, 다른 주제와 섞여 나열하면 안 된다.
  - 예: GPT-5.4 관련(발표, System Card, 추론/CoT, 특정 플랫폼 도입 등)은 **한 블록**으로 연속해 서술하고, Cursor 제품/이슈는 그 다음 블록으로, Claude/Anthropic은 또 다른 블록으로 구분한다.
  - 섹션(1~4) 안에서도 **주제별로 그룹을 나누고**, 각 그룹 내에서만 항목을 나열하라.
- **섹션 형식 (시각적 구분 강화)**: 섹션 제목은 마크다운 헤딩(H3) 문법을 사용하여 다음 형태로 작성하라:
  - 예: `### 1. 에이전틱 코딩 & 프론티어 모델: "부제(따옴표 한 줄)"`
  - 섹션 제목(번호·제목·부제) 다음에는 반드시 한 줄의 빈 줄을 넣어 소개 단락이 다음 줄에서 시작되게 하라.
  - 섹션 제목(마크다운 헤딩)은 각 토픽 블록을 시각적으로 명확히 구분하기 위한 필수 형식이다.
- **섹션 내용 구성**: 섹션 제목 다음에는 한 단락의 소개(2~4문장)를 넣고, 그 아래에 주제별로 묶은 뉴스 항목을 서술형으로 나열하라.
- **항목 형식 (타이틀과 요약의 줄바꿈 명확화)**: 각 뉴스 항목은 다음 형식을 엄격히 따르라.
  - 항목 제목 라인은 `**[제목 또는 출처 요약]:**` 형태로 **독립된 한 줄**에 작성하라.
  - 항목 제목 라인 다음에는 반드시 한 줄의 빈 줄을 넣고, 그 다음 줄에서 2~4문장의 흐름 있는 설명 단락을 시작하라.
  - 설명 단락 안에 필요한 링크는 인라인 `[텍스트](URL)`로만 넣어라.
  - 예:
    - `**[Cursor 2.5+: Cloud Agents with Computer Use]:**`
      
      커서는 2월 24일 클라우드 에이전트 기능을 정식 공개했다. 사용자가 IDE를 종료해도 클라우드 샌드박스에서 에이전트가 수 시간 동안 코드를 수정·테스트하고 PR을 생성한다. (이어서 맥락 1~2문장).
  - 항목은 불릿이 아닌 서술형 단락으로 제공되어야 하며, 각 항목의 설명은 하나의 연속 문단으로 유지하라.
- **섹션 구성(권장 항목)**:
  1. 에이전틱 코딩 & 프론티어 모델 — Cursor, Claude Code, Codex, Composer 등 개발자 도구·에이전트 관련
  2. GitHub & 인프라 트렌드 — CI/CD, 오픈소스, 인프라, 마크다운 자동화 등
  3. 산업별 AX — 트래블, 금융, 제조 등 산업별 AI 적용·경험(AX)
  4. 대한민국 IT Top 10 (맥락화된 뉴스) — 한국 기업·정책·시장 뉴스가 있으면 "주제: 한 줄 요약" 형식으로 최대 10개 나열. 없으면 이 섹션은 "4. 기타 인사이트"로 짧게 처리하거나 생략 가능.

### 4. 금지 사항
- **동일 주제 흩어 쓰기**: 한 제품/이벤트(예: GPT-5.4) 관련 항목을 여러 곳에 나눠 넣거나 다른 주제와 섞어 나열하지 말 것. 반드시 같은 주제끼리 한 블록으로 묶을 것.
- **불릿 나열만 하는 항목**: 각 뉴스는 반드시 `**[제목]:**` 타이틀 라인(독립된 줄) + 빈 줄 + 서술형 문단의 형태로 작성할 것.
- **Insight Gap Analysis, 체크리스트, 권장 액션 목록** 같은 운영형 섹션 추가 금지.
- **참고·원문 (선택 링크)** 처럼 본문 끝에 링크만 나열하는 블록 금지. 링크는 본문 서술 안에만 인라인으로.
- **"요청하시면 … 작성해 드리겠습니다"** 같은 CTA 문구 금지.
- 문서 끝에 `'— 끝 —'` 또는 유사한 마무리 표기나 종결 카드(정보 가치가 없는 단순 마감 문구)는 절대 붙이지 말 것. (이 조항을 반드시 준수하라 — 불필요한 마무리 표기 금지)
- 카드형 마무리 등으로 정보를 희석하지 말고, 모든 섹션과 마지막 문장까지 실질적 정보가 되도록 작성하라.

## 출력
- **마크다운만** 출력하라. 코드 블록이나 JSON 없이, 헤드라인부터 본문·섹션 4까지 연속된 마크다운 본문 한 편.
