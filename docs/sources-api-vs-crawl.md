# X / Threads / Reddit 수집: API vs 크롤링

**현재 프로젝트 기본 전략:** API 대신 **RSS 브릿지 + Reddit RSS + hnrss.org + Firecrawl** 등 비(非) API 소스를 사용한다.  
→ 소스 목록: `config/sources.yaml`  
→ 전략 요약: `docs/sources-non-api-strategy.md`

---

## 크롤링(비 API) 가능 여부

| 플랫폼 | 크롤링 가능? | 비고 |
|--------|--------------|------|
| **X (Twitter)** | **ToS상 금지** | 이용약관에 "X의 서면 허가 없이 스크래핑·크롤링 금지" 명시. 위반 시 계정 정지·법적 조치 가능. 타임라인·검색 등 대부분 로그인 필요해 공개 스크래핑도 제한적. |
| **Threads** | **ToS상 금지** | Meta 정책상 무단 스크래핑 금지. 공식 API 사용 권장. |
| **Reddit** | **위험** | 공개 페이지가 많아 기술적으로는 가능하나, 자동화 스크래핑은 ToS 위반 소지·IP 차단 가능. |

**정리:** API 방식이 규정상·운영상 안전함. 크롤링은 기술적으로 구현 가능하더라도 **개인·비상업 용도에서도 계정/IP 리스크는 사용자 책임**으로 두고, 설정에서 `method: crawl`을 선택할 수 있게 두는 방식 권장.

## 일반 사용자 API 키 발급·사용료 (2025 기준)

### X (Twitter)

- **개발자 계정:** 무료로 생성 가능 (developer.x.com).
- **API 사용:** **무료 티어 없음** (2023년부터 유료 전환).
  - Basic: 약 **$200/월** (월 약 1만 포스트 읽기 등).
  - Pro: 약 **$5,000/월** (월 약 100만 포스트).
  - 사용량 기반: 읽기 약 $0.005/건, 쓰기 약 $0.01/건 등.
- **결론:** 일반 사용자가 “무료로 X API로 수집”하는 것은 사실상 불가. 소량 테스트용 legacy 무료 할당이 있을 수 있으나 공식적으로는 유료.

### Threads

- **공식 API:** Meta for Developers에서 앱 생성·권한 설정. **공개된 사용료 정보 없음** (인프레션 기반 제한 등).
- **서드파티(예: RapidAPI):** 무료 50 req/월, 유료 $19.99/월~(요청 수에 따라).
- **결론:** 일반 사용자도 앱 만들면 공식 API 사용 가능하나, 비용은 Meta 정책·서드파티 요금표 확인 필요.

### Reddit

- **일반 사용자:** 개발자 등록 후 **무료 티어 사용 가능**.
  - OAuth 인증: **분당 약 100 요청**.
  - 비인증: 분당 약 10 요청.
  - 월 약 **1만 요청** 수준 제한.
- **초과 시:** 약 **$0.24/1,000 요청** (2025년 기준).
- **결론:** 일반 사용자도 API 키 발급 가능하고, **소규모 수집은 무료 티어로 가능**. Reddit이 세 플랫폼 중 유일하게 실질적인 무료 API 제공.

## 설정에서 선택 가능하게 (권장)

- 소스별로 **`method: api`**(기본) 또는 **`method: crawl`** 를 두면,  
  - API: 공식/서드파티 API만 사용 (안전, 비용은 플랫폼별).  
  - crawl: Playwright 등 브라우저 자동화 (구현 가능하나 ToS·리스크는 사용자 책임).
- 구현 시 `method`가 없으면 `api`로 간주하고, `crawl`일 때만 크롤링 툴을 사용하도록 분기하면 됨.
