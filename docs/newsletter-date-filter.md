# 기준일 당일 뉴스만 수집하도록 하는 날짜 필터

## 문제(리뷰 결과)

- 레터가 "2026년 3월 2일자의 핵심 AI 뉴스"라고 하면서 실제로는 **2025년 하반기** 발표(Aardvark, Codex GA, Sora 2, Samsung·SK Stargate 등)가 포함됨.
- 즉 **기준일 당일 뉴스가 아니라 과거 발표 총정리** 형태가 되어, "특정일 데일리 브리핑" 요구와 불일치함.

## 원인

1. **수집(collect)에 날짜 필터 없음**  
   피드가 반환하는 "최근 N건"을 그대로 사용해, 기준일(`date_str`)과 무관하게 여러 시기의 항목이 섞여 들어감.
2. **LLM이 입력 목록 외 지식으로 보강**  
   analyze/letter_generate 단계에서 "입력에 없는 뉴스는 쓰지 말 것" 구속이 없어, 모델이 아는 과거 발표를 추가함.

## 수정 사항

### 1. 수집 단계 날짜 필터 (`pipeline/collect.py`)

- **`_published_to_date(published)`**  
  항목의 `published` 문자열(ISO8601 등)에서 **날짜만 추출**. 파싱 실패·빈 문자열이면 `None`.
- **`_filter_items_by_date(items, date_str)`**  
  **기준일(`date_str`, YYYY-MM-DD)에 발표된 항목만** 남김.  
  `published`가 없거나 다른 날짜면 **제외** (크롤 소스는 `published`가 비어 있어 제외됨).
- 수집 직후 `all_items`에 위 필터를 적용해, **당일 발표 뉴스만** 다음 단계(analyze 등)로 전달.

### 2. 스킬 제약 보강

- **`skills/analyze.md`**  
  "입력으로 전달된 항목만 분류·출력할 것. 목록에 없는 뉴스·발표·URL은 절대 추가하지 말 것." 명시.
- **`skills/letter_generate.md`**  
  "아래 items에 있는 항목만 사용할 것. items에 없는 사건·발표·URL은 서술하지 말 것. 모르는 내용을 추론하거나 보강하지 말 것." 명시.

## 동작 정리

- **기준일** = 파이프라인 실행 시 사용하는 날짜(전일 기본값).
- **수집** = 피드·크롤 수집 후, `published`가 **기준일과 같은 항목만** 유지.
- **분석·레터** = 그렇게 남은 항목만 분류·서술하며, **외부 지식으로 항목을 추가하지 않음**.

## 검증

- `python -m pipeline --date 2026-03-02 --force` 실행 시:
  - 수집 후 항목 수가 크게 감소(예: 962 → 52), 기준일 외 항목 제거됨.
  - 생성된 레터는 2026년 3월 2일 당일 커뮤니티·장애·연구 논의 등만 포함, 2025년 하반기 발표는 포함되지 않음.
